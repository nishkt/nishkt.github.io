AIML and Sequence-to-Sequence Models to Build Artificial Intelligence Chatbots: Insights from a Comparative Analysis

this study critically compares the Artificial Intelligence Mark-up Language (AIML), and Sequence-to-Sequence models for building chatbots. In this endeavor, two chatbots were developed to implement each model and were evaluated using a mixture of glass box and black box evaluation, based on 3 metrics, namely, user’s satisfaction, the information retrieval rate, and the task completion rate of each chatbot. Results showed that the AIML chatbot ensured better user satisfaction, and task completion rate, while the Sequence-to-Sequence model had better information retrieval rate.

TORUL (Employment Law Expert System)

TORUL is an expert system I had created to assist Mauritian residents with employment law queries. It utilized a Flask (Python) backend, with a SQLite DB, and a standard HTML,CSS,JS frontend. It had speech recognition and synthesis features using the webspeech API, and used Machine Learning (Sigmoid function) to determine correct responses based on user's queries (either through typing or speaking).

AIML and Sequence-to-Sequence Models to Build Artificial Intelligence Chatbots: Insights from a Comparative Analysis

This was my final year project, where two chatbots were developed for the Middlesex University to help potential/current students, lecturers, and parents gather information about the university. This was run with a Python backend, running on the command line. One used the AIML (Artificial Intelligence) technology, which is a derivation of XML. The other used a Sequence-to-Sequence model that uses RNNs (Recurrent Neural Networks) to predict each word based on the user's query. The Sequence-to-Sequence model used a dataset based on information gathered from university staff, and website resources (FAQs, info, etc).
